OUTPUT_BUFFER(9) Manual Page

NAME

output_buffer - Feed through multiple bits when enable pin is set

SYNOPSIS

loadrt output_buffer [count=N|names=name1[,name2…​]] [personality=P1[,P2…​]]

DESCRIPTION

Feed through up to 32 bit inputs to their outputs when enable set, otherwise outputs will be the value of off-level (default 0).

                          ┌───────────┐
output-buffer.N.in-0 ─────┤  ──[>]──  ├───── output-buffer.N.out-0
                          │   ┌─┘     │
output-buffer.N.in-1 ─────┤  ─┼[>]──  ├───── output-buffer.N.out-1
                          │   ├─┘     │
output-buffer.N.in-2 ─────┤  ─┼[>]──  ├───── output-buffer.N.out-2
                          │   ├─┘     │
                 ... ─────┤   :       ├───── ...
                          │   :       │
output-buffer.N.in-M ─────┤  ─┼[>]──  ├───── output-buffer.N.out-M
                          │   ├─┘     │
              enable ─────┤  ─┘       │
                          │           │
           off-level ─────┤ [0/1]     │
                          └───────────┘

The number of channels are determined by the value of personality.

FUNCTIONS

output-buffer.N

PINS

output-buffer.N.enable bit in

Enable input

output-buffer.N.off-level bit in (default: 0)

Signal level when output is off (enable=0)

output-buffer.N.in-M bit in (M=0..personality)

Inputs

output-buffer.N.out-M bit out (M=0..personality)

Outputs (Follow inputs when enable=1)

EXAMPLES

loadrt output_buffer count=1 personality=8 will create a 8 bit buffer.
