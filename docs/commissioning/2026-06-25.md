## E-stop external-fault polarity — CORRECTED
`F4-02` set to **11**, not 33. Empirically `33` (closed-relay) faults on an OPEN
S3; the R2 NC wiring holds **S3 open when healthy** (e-stop released → R2
energized → R2B2 NC open). So we need fault-on-CLOSED = **11**. Confirmed by test:
with `33`, pressing an e-stop *cleared* Err15 (backwards). `F4-03 = 9` (reset, R2
NO). No cabinet wiring changed.

## Key finding — Colombo is too big for this drive
The 7.5 kW drive on **single-phase** derates to ~3 kW. Driving the 11.8 kW Colombo
up in speed → **Err10 (AC drive overload)** — the drive's own inverse-time
overload, not motor overload. Raising the current trip does NOT help (more current
= Err10 sooner). Cause = magnetizing current of the big motor exceeds the
single-phase-derated drive. **Decision:** use a properly matched (smaller) spindle
later; the VFD is left configured so it's ready when that spindle arrives. Redo
`F1`, `F8-36`, `F0-10/12` for the new spindle then.

## Measurement notes (single-phase artifacts)
- Input is ~**250 V** (not 220): DC bus = 250 × √2 ≈ **354 V** (shown on display
  top, with the 0/freq on the bottom line).
- DMM on the PWM output reads garbage/inflated (354 V, 257 V) — trust the drive's
  output-voltage display (= Hz × 1.27).
- "40 A" on the input ammeter = peak artifact of the pulsed single-phase rectifier
  current; true RMS is a few amps (consistent with <5 A output).
- Loud buzz at low speed = **120 Hz bus-ripple torque pulsation** (inherent to
  single-phase); maps to no fault/alarm; eases with speed.

## Pendant axis-select — FIXED (bad splice) ✅
input-00 read constant FALSE; long diagnosis. Root cause = a **bad splice between
the toggle COM and MPG/B2** (same failure mode as the rack sensor). It put ~kΩ in
the switch's 0 V return, so the closed switch could only pull IN0 to ~3.7 V (B2 sat
at ~0 V, toggle-COM at 3.7 V — the drop across the bad splice = the input's sensing
current × the splice R). Only ~1.3 V across the isolated input (needs ~4 V) → FALSE,
barely moving when pressed. Common polarity was correct all along (per 7I97 manual:
inputs 4–36 V, positive *or* negative common; sinking signal → +5 V common = right).
Remade the splice → IN0 now toggles cleanly (TRUE at rest, FALSE pressed, 8 clean
transitions). Still TODO: HAL cycle-select logic (one button → cycle X/Y/Z) to
source sig-mpg-sel-x/y/z (ned.hal:363, still commented).

## R3 run/stop interlock — PASS ✅ (drives-live test)
Done with **e-stops RELEASED** (drives live; X8 not landed so commands floating —
flagged runaway risk, user proceeded with e-stop in hand; nothing moved). Cycled

## Spindle from LinuxCNC — FULL LOOP PASS ✅ (X8 now landed)
X8 connected, VFD switched to LinuxCNC control: **F0-02=1** (Terminal run from
S1/S2 ← spin-cw/ccw), **F0-03=3** (AI2 speed ← pwmgen.04 via X8). Drove the
spindle by forcing the low-level pins (output-09/10 + pwmgen.04.value/enable;
scale 24000 RPM=10 V). Results:
- **(a) CW 7 s @ ~25 Hz** — spindle ran CW ✅
- **(b) CCW 7 s @ ~25 Hz** — spindle reversed ✅ (guard waits for spin-down before reversing)
- **(c) overload test:** ~50 Hz/30 s sustained **clean** (no fault). Pushed to the
  **150 Hz cap** → **faulted at 36 s** (≈30 s ramp to 150 Hz + ~6 s at speed). The
  fault **propagated to LinuxCNC: `sig-vfd-fault` (input-13) → TRUE**, spindle-running
  → FALSE, drive coasted (no braking on fault). ✅
- **(d) clear:** punched + released the e-stop → R2A2 pulses **S4 reset** → Err
  cleared → **`sig-vfd-fault` → FALSE**. ✅ (No CNC reset output wired; e-stop cycle
  is the clear path.) `F8-18=1` startup-protection blocks any re-spin on the clear.

So LinuxCNC commands run/dir/speed, the drive's fault is read back by the CNC, and
an e-stop cycle resets it. The single-phase-derated drive sustains the no-load
Colombo up to ~50 Hz but overloads near 150 Hz — consistent with it being too small
for this spindle (use a matched spindle for real work; VFD config stays ready).
